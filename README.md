# helm-concurrency

This is some sample of code that causes failures talking to the helm client when using goroutines.

To run this you need to have a working kubectl and a tiller deployed in your K8s cluster.

Before running for the first time you will also need to run a `dep ensure` to install dependencies.

To run the code you can just run `sh test.sh`. This will create a nicolerenee namespace and
deploy 20 hello world releases via helm into this namespace. Once this is created it will port port forward
the tiller to localhost so the go script can talk to the tiller. This then fires off the go test.
The first step is running through and validating all 20 releases exists. The second test does the exact same
check but runs the check in a goroutine. If everything works both should pass. Instead I see something like this:
